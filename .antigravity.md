# 🌌 Neovim Configuration Workspace Rules

Welcome to the active development workspace for this custom **Neovim Configuration**.

## 📂 Project Overview
This repository contains a modular Neovim configuration powered by **Lazy.nvim** and styled with **Catppuccin Mocha**. It is designed to act as a lightweight, high-performance IDE.

*   **Entrypoint**: [init.lua](file:///home/sriram-nithyanandam/.config/nvim/init.lua) (requires options, keymaps, autocmds, and lazy loader).
*   **Settings & Core**: Located in [lua/config/](file:///home/sriram-nithyanandam/.config/nvim/lua/config/) ([options.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/config/options.lua), [keymaps.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/config/keymaps.lua), [autocmds.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/config/autocmds.lua)).
*   **Plugins**: Configured modularly inside [lua/plugins/](file:///home/sriram-nithyanandam/.config/nvim/lua/plugins/).

---

## 🤖 Rules for Agent Behaviors
When assisting the user within this workspace, you **MUST** adhere to the following rules:

### 1. Maintain Modular Architecture
*   **Do NOT** dump new configurations or plugin settings directly into `init.lua`.
*   Every new plugin should be defined in its own file under `lua/plugins/` (e.g., `lua/plugins/my-plugin.lua`).
*   Follow the Lazy.nvim return format: `return { { "author/plugin", opts = {} } }`.

### 2. Preserve User Customizations
*   Check [lua/config/keymaps.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/config/keymaps.lua) and [lua/config/options.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/config/options.lua) before adding new keybindings or options to prevent conflicts.
*   Retain any existing settings unless explicitly instructed by the user.

### 3. Styling & Formatting Guidelines
*   Keep Neovim Lua files clean and formatted.
*   This workspace relies on [conform.nvim](file:///home/sriram-nithyanandam/.config/nvim/lua/plugins/conform.lua) configured with **StyLua** for automatic formatting on save. 
*   Avoid inline formatting scripts; let the file-saving hook handle layout styling.

### 4. Language Server & Tool Integrations
*   Manage language servers, linters, and formatters via [mason.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/plugins/mason.lua) and [mason-tool-installer.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/plugins/mason-tool-installer.lua).
*   If you need to support a new language, update the list in `mason-tool-installer.lua` and add the language setup to [lsp.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/plugins/lsp.lua) and [conform.lua](file:///home/sriram-nithyanandam/.config/nvim/lua/plugins/conform.lua).
